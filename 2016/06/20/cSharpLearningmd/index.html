C# Learning

发表于 2016-06-20   |   分类于 C#   |  

主要内容：关键知识点

关键点

1. cs文件的手动编译

找到csc.exe文件的存放路径（一般位于Framework的安装目录下，注意选择操作系统的版本，如果是64位选择Framework64）并将其配置到系统环境变量的path中以方便手动编译；

（1）记下 .cs文件的路径；

（2）进入cmd；

（3）粘贴需要编译代码的路径；

（4）编译格式：csc 带后缀名的代码文件；

（5）编译成功可以生成可执行的exe文件。

2. 关键问题点

  1. 一个项目只能有一个Main函数作为主入口，当有多个入口函数时需要选择你所需要启动的类名；
    在VS界面：项目——当前项目的属性（xx属性）——应用程序——启动对象（选择你要启动的类）

  2. 用const修饰的常量在声明时不允许使用static关键字，同时必须在声明的时候进行初始化

  3. 字符类型（char）的变量是用 ‘’ 括起来的，如果用 “” 括起来则表示字符串；

  4. C#支持两种预定的引用类型object和string

  5. 应该在开始一个项目之前就计划定义好名称空间的层次结构，一般格式为：CompanyName.ProjectNameSystemSection

  6. C#中的命名约束，公有属性采用Pascal大小写形式命名，如果存在一个等价的私有字段则采用camel大小写形式命名

  7. C#中的结构与类
    （1）结构是值类型，不是引用类型，以内联的方式存储（inline）；
    所以结构中的new关键字与类不同（类中的new关键字是必须进行初始化的引用），只是调用相应的构造函数传入参数来初始化字段；可以不使用new关键字，可以直接定义一个结构类型的对象；

（2）结构不支持继承；

（3）结构总是提供一个无参的默认的构造函数，并且不允许被替换；但是也可以自己定义构造函数；
结构中默认的构造函数隐式的把数值字段初始化成0，把引用类型的字段初始化为null，字段的初始化不能绕过默认的构造函数；

（4）结构是数据项的组合，大多声明为public；

（5）结构和其他数据类型一样，使用之前所有的元素必须进行初始化；

（6）如果把结构作为参数传给另外一个结构，则所有内容被复制，对于类来说，只是复制引用；

（7）当结构作为参数传给方法时，应把它作为ref参数进行传递，这样只传递结构在内存中的地址，可以节省性能，此时相应的调用方法可以改变结构的值；

（8）结构不可以继承但是可以访问System.object方法；

（9）静态类不允许创造实例，如果一个类只包含静态的方法和属性，该类就是静态的，通过类名调用该类；

3. 语法点

  1. C#不支持私有继承，因此在基类名上不存在public和private限定符

  2. 成员字段和静态函数都不能声明为vitual

  3. C#允许把类和函数声明为abstract, 抽象类不能实例化, 而抽象函数不能直接实现, 必须在非抽象的派生类中重写

  4. C#允许把类和方法声明为sealed，表示不能被继承的类和不能被改写的方法，要在方法或属性上使用sealed关键字,必须先从基类上把它声明为要重写的方法或属性；
